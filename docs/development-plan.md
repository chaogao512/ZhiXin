# 知新 (ZhiXin) 开发计划

> 本文档将整个项目拆分为可执行的开发阶段，按顺序推进。
> 每个阶段包含具体任务、技术要点和完成标准。

---

## 开发总览

```
Phase 1 ─→ Phase 2 ─→ Phase 3 ─→ Phase 4 ─→ Phase 5 ─→ Phase 6
 基础设施     核心闭环       学生端         家长端         教师端       联调与部署
```

| 阶段 | 内容 | 预计步数 |
|------|------|---------|
| **Phase 1** | 服务端与客户端基础设施搭建 | 10 步 |
| **Phase 2** | 拍照收集 → OCR → LLM 分析核心闭环 | 8 步 |
| **Phase 3** | 学生端全部功能 | 6 步 |
| **Phase 4** | 家长端全部功能 | 6 步 |
| **Phase 5** | 教师端全部功能 | 6 步 |
| **Phase 6** | 多端联调、性能优化、部署 | 4 步 |

---

## Phase 1：基础设施搭建

### 目标
服务端可运行并响应 API 请求；客户端可编译运行，完成导航框架和网络层。

### 1.1 服务端：数据库模型与迁移

| 项目 | 说明 |
|------|------|
| **任务** | 实现所有 SQLAlchemy 模型的完整定义，建立表关联 |
| **文件** | `server/app/models/user.py`、`server/app/models/mistake.py`、`server/app/models/class_.py` |
| **关键表** | `users` / `classes` / `class_members` / `parent_student` / `subjects` / `chapters` / `mistakes` / `analysis_results` / `practice_assignments` / `practice_submissions` |
| **完成标准** | `alembic init` 初始化迁移，`alembic upgrade head` 成功建表 |

### 1.2 服务端：Pydantic Schema

| 项目 | 说明 |
|------|------|
| **任务** | 实现所有 API 请求/响应的 Pydantic 模型 |
| **文件** | `server/app/schemas/` 下各模块 |
| **包含** | 认证 / 错题 / 分析 / 班级 / 练习 / 报告 六类 |

| **方案** | Docker 容器化 + docker-compose（FastAPI + PostgreSQL + Redis）/ 或直接部署到云服务器（阿里云/腾讯云） |
| **完成标准** | 服务端可稳定运行，API 响应正常 |

---

## 开发顺序建议

```
Phase 1 ──────────────────────────────────────────
         └──── 服务端优先 ──── 客户端并行 ────

Phase 2 ──────────────────────────────────────────
         └──── 客户端优先 ──── 服务端配合 ────

Phase 3/4/5 ──────────────────────────────────────
         └──── 按角色顺序开发，每个角色的服务端先行 ────

Phase 6 ──────────────────────────────────────────
         └──── 全量集成 ──── 修复 ──── 上线
```

### 推荐开发节奏

1. 每个 Phase 开始前，先实现该 Phase 涉及的服务端 API
2. 服务端 API 完成并用 Postman 验证后，开始客户端 UI
3. 每个子任务完成后进行本地联调
4. 每个 Phase 完成后进行该阶段的全流程测试
