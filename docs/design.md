# 知新 (ZhiXin) — 设计文档

> 版本: v0.2 · 日期: 2024-03-15

---

## 1. 项目概述

知新是一款面向中小学生、家长和教师的错题收集管理与智能分析系统。学生通过拍照或手动录入方式收集错题，经由服务端大模型（LLM）进行深度分析，生成错误原因解析、薄弱知识点定位、相似题推荐及个性化学习建议。系统覆盖 iOS、iPadOS 和 macOS 三端，根据用户角色呈现差异化功能界面。

### 1.1 核心理念

「温故而知新」—— 通过回顾和深度分析错题，发现知识盲区并获得新的认知提升。

## 2. 用户角色与权限

| 角色 | 客户端 | 功能范围 |
|------|--------|---------|
| 学生 | iOS / iPadOS | 拍照录入错题、查看错题本、查看 LLM 分析、做推荐练习 |
| 家长 | iOS / iPadOS / macOS | iOS/iPadOS: 查看孩子学习概览与报告; macOS: 完整管理界面 |
| 教师 | iOS / iPadOS / macOS | 班级管理、学生错题查看、全班错题统计分析、布置练习 |

### 2.1 角色关系模型（学校组织模式）

```
教师 ──创建──→ 班级 ←──加入── 学生
                           │
                       绑定家长
                           │
                         家长
```

- 教师创建班级 → 生成班级邀请码
- 学生通过班级码加入
- 家长通过学生账号绑定（一个学生可绑定多个家长）
- 教师可管理班级内所有学生和家长


## 5. 核心业务流程

### 5.1 错题收集与 AI 分析流程

```
学生拍照 ──→ 上传服务端 ──→ OCR 识别 ──→ 保存 Mistake
                                              │
学生选择学科/章节 ──→ POST /analyze ──→ BackgroundTasks
                                              │
                                     LLM 调用分析 (3-30s)
                                              │
                                    WebSocket 推送结果
                                              │
                                   Swift Charts 可视化展示
```

### 5.2 离线模式

```
拍照 ──→ 存本地 SwiftData (syncing: pending)
       ──→ 存照片到沙箱
       ──→ 显示"待同步"标记

联网检测到 → 批量上传 → 更新本地状态 → 同步分析结果
```

```
系统角色：中学教育 AI 助手，擅长知识点诊断和错题分析。

输入：
- 学科
```

## 9. 安全与隐私

- Apple ID / 微信 OAuth 认证，服务端不保存密码
- 学生照片和错题数据仅对关联的家长和教师可见
- 图片传输使用 HTTPS 加密
- 遵循中国未成年人数据保护相关法规

---

## 10. 性能考虑

| 场景 | 方案 |
|------|------|
| LLM 调用延迟 (3-30s) | BackgroundTasks + WebSocket 异步推送 |
| 大照片上传 | 客户端压缩后上传，支持断点续传 |
| Swift Charts 大数据点 | 后端聚合降采样，客户端渐进式加载 |
| JSON 解析 | Stream 式解码，避免大 JSON 一次性加载 |
| 离线同步 | 队列 + 指数退避重连 |

## 12. 详细需求规格

### 12.1 学生角色需求

#### 12.1.1 冷启动流程

| ID | 需求 | 说明 |
|----|------|------|
| S-C001 | 独立使用 | 新学生登录后，不强制加入班级即可正常使用全部个人功能（拍照收集、分析、练习） |
| S-C002 | 绑定班级 | 用户可在「设置」中随时搜索班级码加入班级，加入后教师端可见该学生数据 |
| S-C003 | 班级码加入 | 输入 6 位班级邀请码即可加入对应班级，加入后可更换班级 |

#### 12.1.2 拍照收集流程

| ID | 需求 | 说明 |
|----|------|------|
| S-P001 | 多张拍照 | 支持单次拍照一张或多张照片，覆盖一道题、多道题、整个卷面或试卷所有页面 |
| S-P002 | 本地暂存 | 照片拍完后优先本地存储，不要求立即联网上传 |
| S-P003 | 错题标记 | 提供三种并行方式标记错题位置/信息：① 圈画（手指或 Apple Pencil 在照片上圈出）② 语音口述（录音转文字提示）③ 文字输入 |
| S-P004 | 学科选择 | 拍照时可选择学科（数学/语文/英语/物理/化学/生物/历史/地理/政治），也可跳过让 LLM 自动识别（混合模式） |
| S-P005 | 批量上传 | 有网络时，本地暂存的所有照片+标记信息批量上传至服务端 |
| S-P006 | 后台处理 | 服务端 LLM 进行图片扫描 → OCR 识别 → 错题提取 → 分类归档 |

#### 12.1.3 确认归档流程

| ID | 需求 | 说明 |
|----|------|------|
| S-V001 | 智能推荐+快速确认 | 高置信度的错题自动归档，低置信度标记为「待确认」让学生重点关注 |
| S-V002 | 全部展开查看 | 学生可随时展开查看本次上传的全部识别结果 |
| S-V003 | 逐题编辑 | 每道题的 OCR 文本、学科、章节、学生答案、正确答案均可手动修改 |
| S-V004 | 手动录入 | 支持完全不拍照，直接手动输入题目文本、学生答案、正确答案来创建错题 |

#### 12.1.4 错题本浏览

| ID | 需求 | 说明 |
|----|------|------|
| S-L001 | 智能排序 | 按 LLM 推荐复习顺序排列（综合知识点掌握度和遗忘曲线策略） |
| S-L002 | 多维筛选 | 按学科、章节、错误类型（粗心/知识点/审题不清）、时间范围、分析状态筛选 |
| S-L003 | 状态标识 | 每条错题显示「已分析/待分析/分析中」「已掌握/待复习」状态标签 |
| S-L004 | 搜索 | 支持按题目文本关键词搜索错题 |

#### 12.1.5 错题详情与 AI 分析展示

| ID | 需求 | 说明 |
|----|------|------|
| S-D001 | 原题照片 | 展示原始照片，支持缩放查看 |
| S-D002 | LLM 分析卡片 | 展示错误原因、正确解题步骤、薄弱知识点列表 |
| S-D003 | 掌握度评分 | 用进度条或星级展示 LLM 对该知识点掌握度评估 (0-100%) |
| S-D004 | 相似题推荐 | 展示 LLM 推荐的 3-5 道相似练习题（含答案和解析） |
| S-D005 | 学习建议 | LLM 生成的针对性复习建议文本 |
| S-D006 | 个人备注 | 学生可添加/编辑文字备注 |

#### 12.2.8 复习拍照检查（iOS + iPadOS）


### 12.3 教师角色需求

#### 12.3.1 核心定位：AI 分析策略设计者

教师端的核心价值不仅是查看分析结果，更是通过自然语言与后端大模型交互，**设定错题分析的维度和策略**，即：

- 定义从哪些维度解析学生上传的错题（按知识点/题型/难度等）
- 设定分析报告的格式和内容结构
- 定制复习材料生成的策略（出题数量、难度分布、题型配比）
- 与学生端、家长端协同制定复习计划

#### 12.3.2 AI 分析策略配置

**交互方式：表单 + 自然语言混合模式**

| ID | 需求 | 说明 |
|----|------|------|
| T-C001 | 基础表单配置 | 系统提供结构化的配置表单，教师可设置：分析字段开关、分类标签体系、输出格式 |
| T-C002 | 自然语言扩展 | 在表单基础上，教师可通过自然语言补充复杂规则。例如：「把数学错题按知识点细分，每个知识点出 3 道易、2 道中、1 道难题」→ LLM 理解后转化为配置 |
| T-C003 | 配置维度全覆盖 | 可配置项包括：① 分析输出字段 ② 练习生成策略（数量/难度/题型）③ 分类标签体系（自定义知识点树）④ 分析触发条件 ⑤ 复习提醒策略 ⑥ 报告推送规则 |
| T-C004 | 配置预览与保存 | 每种配置修改后，教师可预览效果（用样本数据模拟），满意后保存生效 |
| T-C005 | 多套配置切换 | 教师可为不同班级/学科保存多套配置方案，随时切换 |

#### 12.3.3 AI 策略配置对话示例

```
教师：这学期我们重点抓二次函数和几何证明，帮我调整分析策略

AI：好的，已调整以下配置：
- 「二次函数」和「几何证明」知识点权重提升至 2 倍
- 涉及这两个知识点的错题自动生成 6 道变式练习
- 分析报告中增加「知识点关联性」字段
- 每周一推送这两个专题的薄弱点汇总报告

需要预览效果吗？
```

### 13.2 跨角色数据流

```
                    ┌──────────────┐
                    │   教师端      │
                    │  • 设定策略   │ ←── AI 分析策略配置
                    │  • 布置练习   │ ←── 分发到学生
                    │  • 查看分析   │ ←── 读取全班+个人数据
                    │  • 批注留言   │ ──→ 推送到学生+家长
                    └──────┬───────┘
                           │ 策略配置影响分析行为
                           ▼
┌──────────┐     ┌───────────────┐     ┌──────────┐
│ 学生端    │ ──→ │   服务端      │ ←── │ 家长端   │
│ 拍照收集  │     │  LLM分析引擎   │     │ 查看报告 │
│ 确认归档  │     │  OCR识别       │     │ 复习拍照 │
│ 练习作答  │     │  存储+统计     │     │ 复查分析 │
│ 查看分析  │     │  WebSocket推   │     │ 批注查看 │
└──────────┘     └───────────────┘     └──────────┘
```

**核心数据流路径：**

1. **学生拍照 → LLM 分析 → 分析结果入库 → 学生确认 → 家长可查看 → 教师可查看**
2. **教师配置策略 → 影响 LLM 分析格式 → 影响学生/家长端展示**
3. **教师布置练习 → 学生收到并作答 → 家长拍照复查 → 复查结果反馈**
4. **教师批注留言 → 推送给学生和家长**
5. **学生离班 → 个人信息脱敏 → 知识点汇入通用易错知识库 → 增强 LLM 共性分析能力**

### 13.3 关键约束与边界

| 类别 | 规则 |
|------|------|
| **数据可见性** | 学生：仅见自己 / 家长：仅见绑定孩子 / 教师：见班级所有学生 |
| **数据所有权** | 错题数据归学生所有，教师/家长仅在其关联范围内可查看 |
| **离班数据** | 学生离班后，教师不可再查看该生个人信息；该生的错题知识点经脱敏后汇入「通用易错知识库」，供 LLM 分析时参考，提升共性薄弱点识别准确率 |
| **删除权限** | 学生可删除自己的错题；家长不可删除；教师不可删除 |
| **编辑权限** | 学生可编辑自己的错题和家长备注；家长只读；教师可加批注不可改原数据 |
| **账号独立性** | 一人一号，不可同时拥有多个角色。如需切换角色需用不同账号 |
| **班级归属** | 学生同一时间可加入多个班级；教师可创建多个班级 |
| **离线限制** | 学生拍照可离线，确认归档需联网；家长/教师大部分操作需在线 |
| **推送通知** | 仅家长端接收推送；学生端和教师端不设推送通知 |
